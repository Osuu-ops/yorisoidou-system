# OP-3｜Scope Guard（非干渉ガード）｜許可/例外/禁止の一次根拠固定
## 目的
ログ・zip・想定外ファイル混入を自動排除し、システム側の変更がビジネス領域に混入・汚染しない構造を維持する。
## 方針（一次根拠として固定すべき内容）
- 非干渉対象ファイルの分類
  - 許可（Allow）
  - 例外（Exception / Lock）
  - 禁止（Deny）
- 自動除外テスト（混入時に検知し、成功時のみ次工程）
## 失敗時の契約（STOP）
- 失敗検知時は即停止し、成功時のみ次工程へ進む（ガード漏れ防止）
- checks failure 検知後に merge へ進み得るフローは契約違反として禁止
