業務 UI ― 水道修理受付 UI 適用仕様

1. 対象仕様の概要

本 UI_spec_水道修理受付 は、
エンドユーザーが水道修理を依頼するための業務 UIに対して、
UI_PROTOCOL に定義された UI 統治・意味仕様を
水道修理受付という業務文脈に適用した結果を記述するものである。

本書は以下を満たす。

本書は UI_PROTOCOL に従属する

本書は 業務 UI 専用の派生文書である

本書は UI_spec_MEP（管理 UI）とは責務を分離する

本書は UI の原則・状態定義を再定義せず、参照に留める

2. 業務 UI の役割（本業務）

水道修理受付 UI の役割は以下に限定される。

修理依頼に必要な情報を 迷わせずに入力させる

入力途中・未完了状態を 不安なく継続可能にする

受付状況（送信中／完了）を 誤認させない

修理可否・料金・対応可否等の 判断を行わない

本 UI は 受付窓口であり、
業務判断・仕様化は MEP 側の責務とする。

3. 画面構成（水道修理受付 UI）
3.1 画面種別

入力画面（主画面）

修理依頼に必要な情報を入力する画面

単一画面構成を基本とし、必要に応じて段階分割を許容する

確認画面

入力内容を送信前に確認する画面

入力画面へ戻る操作を提供する

完了画面

受付が完了したことを示す画面

次に取るべき行動を簡潔に示す

4. 入力項目（水道修理受付）
4.1 入力項目一覧

発生内容（自由記述）

例：水が止まらない、詰まっている 等

発生場所

例：キッチン／洗面所／トイレ／屋外

発生箇所の詳細（任意）

希望訪問日時（任意）

氏名

連絡先（電話番号等）

住所（市区町村まで可、詳細は任意）

4.2 入力補助の原則

専門用語は使用しない

プレースホルダーは 正解例ではなく記述方向を示す

未入力エラーは 責めない文言で示す

5. 操作順序（水道修理受付）

入力画面を表示

修理内容を入力

確認画面で内容を確認

送信処理を実行

完了画面を表示

※ 操作順序は強制ではないが、本順序を基準とする。

6. 状態と表示の関係
6.1 処理中表示

送信処理中は、処理中であることを明示する

二重送信を誘発する操作は表示しない

6.2 完了表示

「受付が完了した」事実のみを示す

修理確定・料金確定等の断定表現は行わない

7. 表示文言（水道修理受付）
7.1 入力開始時

水道修理のご依頼内容を入力してください。
途中で中断することもできます。

7.2 入力中

入力内容はまだ送信されていません。
必要に応じて修正してください。

7.3 処理中

送信処理を行っています。
完了するまでお待ちください。

7.4 入力不備時

入力内容に不足があります。
該当箇所をご確認ください。

7.5 完了時

受付が完了しました。
内容を確認のうえ、後ほどご連絡します。

8. MEP との責務境界

本 UI で入力された情報は、業務入力パッケージとして MEP に渡る

受け渡し先は MEP のカルテ入力のみとする

本 UI は master_spec を直接編集しない

判断・整理・生成は MEP 側の責務とする

9. 本書の更新ルール

本書は 水道修理受付 UI の仕様変更時にのみ更新する

MEP 内部仕様の変更により更新されることはない

UI_PROTOCOL の変更を伴う修正は行わない

UI 実装は、本書との差分として GitHub で管理する
