# SYSTEM_GUARD_AXIOMS
（SYSTEM 機械判定ガード公理定義）

本書は、  
MEP における **SYSTEM 側で機械的に検出可能な違反条件**を定義する。

本書は、

- 思想を定義しない  
- 判断主体を定義しない  
- フェーズを定義しない  

本書は、  
**Yes / No の機械判定のみ**を目的とする。

本書は、  
MEP_PROTOCOL に定義された思想および  
system_protocol に定義されたフェーズ制御に  
**一切介入しない。**

---

## 公理 G-01：行削除禁止

対象文書において、  
**既存行の削除が1行でも検出された場合**、  
SYSTEM は **違反（NG）** と判定する。

---

## 公理 G-02：行置換禁止

対象文書において、  
**同一行番号で内容が異なる行が検出された場合**、  
SYSTEM は **違反（NG）** と判定する。

（意味同一・表現改善・誤字修正は考慮しない）

---

## 公理 G-03：diff 非経由変更禁止

diff を伴わない変更が検出された場合、  
SYSTEM は **違反（NG）** と判定する。

---

## 公理 G-04：read-only 領域書込み禁止

以下の read-only 領域に対する書込みが検出された場合、  
SYSTEM は **違反（NG）** と判定する。

- X（最新確定仕様）
- S（最新確定成果物）
- Canon / Protocol / Master 文書

---

## 公理 G-05：違反検知後の自動継続禁止

SYSTEM が **違反（NG）** を検知した後に、  
自動処理・自動遷移・自動生成が試みられた場合、  
SYSTEM は **違反（NG）** と判定する。

---

## 公理 G-06：保存条件違反禁止

SYSTEM による **OK 判定が成立していない状態**で、  
X または S への保存が検出された場合、  
SYSTEM は **違反（NG）** と判定する。

---

## 公理 G-07：境界違反入力禁止

API 実装時において、  
以下の違反が検出された場合、  
SYSTEM は **違反（NG）** と判定する。

- read-only 入力の再生成
- candidate を確定扱いする行為
- 会話文脈を判断根拠として使用する行為

---

## 付記

- 上記いずれか **1 つでも成立した場合**、  
  SYSTEM は **違反（NG）** を返す。
- SYSTEM は、  
  理由説明を行わない。
- SYSTEM は、  
  修正案を提示しない。
- SYSTEM は、  
  判定結果を **判断・遷移・合意に変換しない。**

本書は、  
**SYSTEM における純粋な機械判定ガード条件**としてのみ効力を持つ。
