MEP Form UI ― UI 適用仕様（今回の master_spec 用）
1. 対象 master_spec の概要

本 UI_spec は、
現在適用対象となる master_spec に基づき、
UI_PROTOCOL に定義された UI 統治・意味仕様を
今回の業務仕様に対して具体的に適用した結果を記述するものである。

本書は UI_PROTOCOL に従属する

本書は 特定の master_spec 専用の派生文書である

本書は UI の原則・状態定義を再定義しない

2. 画面構成（今回の業務）

本業務における UI 画面構成は、以下とする。

2.1 画面種別

HOME
作業開始前および状態初期化後の基点画面

カルテ画面
業務の背景・前提・補足情報を入力する画面

マスタ画面
業務内容・定義・条件・構造を入力する主画面

コード画面
生成結果（API 定義／データ構造／UI ロジック）を表示する画面

※ 画面種別は UI_PROTOCOL に定義された原則に従う。

4. 入力順序（今回の業務）

本業務における標準的な入力順序は以下とする。

HOME 画面で作業を開始

カルテ画面にて背景・補足情報を入力（任意）

マスタ画面にて業務内容を入力（必須）

一時保存を行う

監査・生成等の処理を実行

結果を確認

※ 入力順序は強制ではないが、
本順序が最も混乱が少ない。

5. 条件付き表示・非表示
5.1 コード画面の表示条件

コード画面は、生成処理が開始された後に表示される。

未生成状態では、コード画面は表示されない。

5.2 操作ボタンの表示条件

操作ボタンの表示は、
UI_PROTOCOL に定義された状態 × 操作対応表に従う。

本 UI_spec では、
追加の例外的表示条件は定義しない。

6. 画面遷移フロー（今回限り）
6.1 新規作成時の遷移

新規作成操作により、

UI 状態は「未開始」に初期化される

表示画面は HOME に戻る

6.2 編集時の遷移

編集操作により、

カルテ画面またはマスタ画面が表示される

画面を開くこと自体は UI 状態を変更しない

6.3 処理実行時の遷移

保存・監査・生成等の処理実行により、

UI 状態は「処理中」に遷移する

処理結果に応じて、

完了

業務エラー

システムエラー
のいずれかに遷移する

7. 業務特有の UI 文言

本業務において使用する UI 表示文言は以下とする。

7.1 新規作成時
新しい作業を開始しました。
内容の入力を始めてください。

7.2 編集中
内容を編集中です。
必要に応じて一時保存してください。

7.3 処理中
処理を実行しています。
完了するまでお待ちください。

7.4 業務エラー時（監査 NG）
内容に不備があります。
修正してください。

7.5 システムエラー時
処理に失敗しました。
キャンセルして最初からやり直してください。

7.6 完了時
処理が完了しました。
次の作業を選択してください。

8. 本書の更新ルール

本 UI_spec は、対象 master_spec が変更された場合にのみ更新する。

UI_PROTOCOL の変更を伴う修正は行わない。

UI 実装（index.html）との差分は、本書を正とする。
