master_spec と UI判断の条件（フルリビルドで全コード一式を生成）を満たすために必要な前提情報が、このチャット入力内に存在しません。master_spec は『命名を勝手に追加・改名しない』『仕様外推測禁止』を要求しているため、以下が欠落した状態では CONFIG.gs / main.gs / HTML 群を“正しく”確定できず、生成すると高確率で既存台帳/Drive構造/外部連携を破壊します。

不足している必須情報（推測禁止のため質問します）:
1) activeBrand / brandProfiles の実データ
   - activeBrand の値
   - brandProfiles 配下の ids（cuPrefix/upPrefix/orderPrefix/odPrefix などの実値）
   - brandProfiles 配下の sheets（実際のスプレッドシート内のシート名マッピング）
   ※ master_spec は『prefix を固定値で書く』ことを許していません（例示はあるが決定値ではない）。

2) CONFIG.storageProfiles の実データ
   - 年度フォルダ切替ルール（年度＋枝番）に使うドライブフォルダID
   - OrderID/ 以下の実保存先ルート（どの共有ドライブ/マイドライブか）
   - photos/videos/logs/archive の各親フォルダID

3) 外部連携の設定値（コード内に必須）
   - Todoist API（token / webhook検証用secret等、実運用の方式）
   - ClickUp API（token / teamId / spaceId / listId など “どこへタスク作成するか” の確定値）
   - ChatGPT API（キー、モデル名、呼び出し可否フラグ）
   - Geolonia API（キー、エンドポイント）
   ※ master_spec は「任意機能」として ClickUp 通知を許容しますが、生成コード内に“無効化の正式スイッチ”や“送信先”がないと Runtime監査/通知仕様を満たせません。

4) 既存スプレッドシートの前提
   - 実在する年度Orderシート名（例：Order_2025 なのか Order_YYYY という実名なのか）
   - Archive_Order_YYYY の実名
   - logs/system の保存が Drive か Sheet か（master_spec は Drive フォルダを定義するが、ログのメタデータ行をどのシートに持つか未記載）

5) Webアプリ公開形態
   - doGet を使う WebApp URL 運用か、Spreadsheet UI（showDynamicForm）専用か
   - 実行権限（実行ユーザー/アクセスユーザー）

上記が揃えば、master_spec の範囲内だけで、CONFIG.gs / main.gs / util群 / RuntimeSelfTest / 全HTML（UF01/UF06/UF07/UF08/FIX/DOC/OV01/OV02）を、命名・ID体系・ステータス・住所保持・checkEnvironment() 必須・logs/system 記録・maintenanceMode ブロック・Runtime監査（expected/unexpected）まで含めてフル生成できます。
